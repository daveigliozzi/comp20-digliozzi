Security Assessment of Game Center

Introduction

Ming Chow Industries hired me to evaluate the security of the web application Game Center. Game Center was developed to take in data from the new, very popular game 2048, developed by Gabriele Cirulli, and use it to create a High Score leader board. Unfortunately, there are vulnerabilities in Game Center that make it unacceptable as a web application.


Methodology

To find vulnerabilities in Game Center, I first conducted what is referred to as “black box” testing. Black box testing involves using tools such as proxies to try to modify code or gain access to the information in the database. I used the proxy Burp Suite for this phase. After black box testing, I went into the actual code to look for vulnerabilities myself.


Abstract of Findings

I have found three flaws in the Game Center system. The flaws revolve around code that was built to function in one way, and disregards non-ideal situations. None of the flaws are unfixable and have relatively simple resolutions.


Issues Found

Cross-Site-Scripting (XSS)

Location:

I found this vulnerability in the app.js file in the app.get function on line 15.

Severity of issue:

High - with XSS, attackers can enter in their own code into the HTML file in the form of a script tag. This code can execute functions that put the application and the data at risk.

Description:

I found this vulnerability by using a proxy to change the data sent to the server from the client-side. I entered in a script tag with code that was executed through the web application Game Center.

Resolution:

This issue can be resolved by filtering the data sent from the client. This can be done in a myriad of ways, such as using an if statement to look for certain signs that the data from the user may be a script tag of some sort.


Adding Non-existent Database Fields

Location:

I found this vulnerability in the app.js file.

Severity of issue:

Low - This can possibly fill up the database, but there is no serious repercussion of extra database fields in the JSON file since nothing in the code allows it to come through on the web application.

Description:

I discovered by looking through the code for app.js that as long as the input has the fields specified, there can be other fields added to the document in the database.

Resolution:

This issue can be resolved by building an object from the fields the developer wants.


Filling Up the Database

Location:

I found this vulnerability in the app.js file.

Severity of issue:

High - Not only can attackers exploit this vulnerability, any user can exploit this vulnerability by simply playing the game too much. The database only has 5 MB of storage and cannot handle too much data.

Description:

I found this issue by looking through the mongoDB database specifics. This is less of a code issue and more of an implementation issue.

Resolution:

The developer can resolve this issue by getting a bigger database to store data.


Conclusion

The current vulnerabilities in Game Center make the web application unfit for use. However, the vulnerabilities are all easily fixed once they were found.
